# AssetTrail

AssetTrail is a static personal asset tracker published with GitHub Pages.

Live app: https://yjmoonn.github.io/assettrail/

## Use

1. Open the live app.
2. Sign in with Google.
   - Before sign-in: the status shows `Cloud ready`.
   - After sign-in: the status shows `Cloud: your-email`, and `Logout`/`Sync` are shown.
3. Add assets with one of four types: `KRX`, `US`, `CASH`, `MANUAL`.
   - Assets with the same market ticker can be tracked separately by account.
4. Press "조회 기록 저장" when you want to record a portfolio snapshot.
5. Use the retirement simulator presets or edit the assumptions directly.

## Price Updates

Prices are generated by GitHub Actions once per day at 06:30 KST.

- Korean stocks/ETFs/ETNs: the workflow generates the full KRX-listed price book automatically. ETF/ETN codes may include letters, such as `0092B0`.
- US stocks/ETFs: base symbols can still live in `tickers.json`, and signed-in users automatically add new US symbols to the shared Firestore price request list when they save US assets. US ticker names are generated automatically from Nasdaq Trader symbol lists.
- The workflow writes `prices.json` with prices, symbol names, product kinds, and `fx.USDKRW`.
- The app reads `prices.json` and calculates value and gain/loss for `KRX` and `US` assets. US prices and average prices are treated as USD and converted to KRW with `fx.USDKRW`.
- `CASH` and `MANUAL` assets use only the manual valuation amount.
- The portfolio panel summarizes account, product kind (`STOCK`/`ETF`/`ETN`), and domestic/overseas exposure.

Example:

```json
{
  "KRX": [],
  "US": ["AAPL"]
}
```

If a US market asset shows "가격 대기", sign in, save/sync the asset, then rerun the workflow or wait for the next daily run. KRX assets are matched against the automatically generated KRX price book. The deployed app also shows price generation errors from `prices.json`.

## Sync

Firebase Auth and Firestore sync assets, history snapshots, and retirement settings under each signed-in user's own document.

Required Firebase Auth authorized domain:

```text
yjmoonn.github.io
```

## Operations

- GitHub Pages deploys from `.github/workflows/deploy-pages.yml`.
- The same workflow generates `prices.json` on push, manual dispatch, and the daily schedule.
- If prices fail, check the latest "Deploy GitHub Pages" workflow run in GitHub Actions.
- Keep personal portfolio data restricted to signed-in users only:

```text
users/{uid}/financeData/primary
```

- The shared `priceRequests/us` document is publicly readable so GitHub Actions can collect requested US tickers, but only signed-in users can write to it.

## Security Checklist

- Do not commit `.env`, service account JSON, private keys, or Firebase Admin credentials.
- Keep Firebase Auth authorized domains limited to the deployed domains you use.
- Restrict the Firebase web API key by HTTP referrer in Google Cloud when the app domain is stable.
- Public Firebase web config is expected for browser apps; Firestore rules are the data boundary.
