# AssetTrail 디자인 리뷰 가이드

UI, 레이아웃, 내비게이션, 시각 토큰을 바꿀 때 이 문서를 사용한다. 목표는 코드만 보는 것이 아니라 실제 렌더링된 앱을 확인하는 것이다.

## 로컬 미리보기

저장소 루트에서 정적 앱을 실행한다.

```sh
python3 -m http.server 4178
```

아래 주소를 연다.

```text
http://localhost:4178/
```

## 확인할 화면 폭

최소한 아래 폭을 확인한다.

| 상황 | 폭 |
|---|---:|
| 넓은 데스크톱 | 1440px |
| 좁은 데스크톱 | 1280px |
| 좁은 모바일 | 390px |
| 큰 모바일 | 430px |

밀도 높은 표, 상단 내비게이션, 하단 내비게이션, 카드, 모달, 차트를 바꿨다면 데스크톱과 모바일을 모두 확인한다.

## 공통 확인 항목

- 텍스트와 컨트롤이 겹치지 않는다.
- 버튼 라벨이 모든 화면 폭에서 읽힌다.
- 모바일 터치 대상이 누르기 쉽다.
- 카드, 패널, 배지, 버튼의 radius와 간격 토큰이 일관된다.
- 비어 있음, 로딩, 로그아웃, 로그인 상태가 모두 이해 가능하다.
- 변경한 흐름에서 포커스 이동과 키보드 접근이 유지된다.
- 해시 기반 내비게이션과 브라우저 뒤로/앞으로 이동이 유지된다.
- 장식 스타일이 금융 데이터를 읽는 데 방해되지 않는다.

## 화면별 체크리스트

### 대시보드

- 요약 지표가 무엇이 바뀌었고 무엇을 봐야 하는지 답한다.
- 데스크톱 레이아웃이 공간을 쓰되 시끄럽지 않다.
- 모바일 레이아웃이 자연스럽게 쌓이고 하단 내비게이션과 충돌하지 않는다.
- 최근 기록이 올바른 자산명 또는 일지명을 보여준다.
- 포트폴리오 구성 표시가 작은 화면에서도 읽힌다.

### 자산

- 원장 행의 텍스트와 액션이 잘리지 않는다.
- 추가, 수정, 매도, 삭제, 일지 액션에 접근할 수 있다.
- 시장가격 자산과 수동평가 자산의 평가 기준이 명확하다.
- `가격 대기` 같은 가격 누락 상태가 보이고 이해 가능하다.

### 일지

- 매수, 매도, 리밸런싱, 관찰 기록이 구분된다.
- 자산 액션과 직접 내비게이션 모두에서 입력 폼에 접근할 수 있다.
- 긴 투자 이유, 리스크, 복기 텍스트가 레이아웃을 깨지 않는다.

### 포트폴리오

- 계좌, 상품 유형, 국내/해외 비중을 비교하기 쉽다.
- 목표 비중 차이가 방향과 크기를 명확히 보여준다.
- 차트가 앱의 글꼴과 색상 토큰을 사용한다.

### 목표

- 히스토리와 은퇴 시뮬레이터 컨트롤이 모바일에서도 읽힌다.
- 숫자 입력, 가정, 결과 카드가 서로 붐비지 않는다.
- 저장된 시나리오나 히스토리 상태가 화면 이동 후에도 명확하다.

### 설정

- 클라우드 동기화, 가격 상태, 가져오기/내보내기, 운영 액션이 차분하게 묶여 있다.
- 위험한 작업은 시각적으로 구분되고 명확한 확인을 요구한다.
- 로그인, 로그아웃, 동기화, 내보내기, 가져오기 컨트롤이 혼란스럽게 중복되지 않는다.
