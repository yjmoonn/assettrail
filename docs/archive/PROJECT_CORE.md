# AssetTrail 핵심 정리

AssetTrail은 개인 자산을 한 화면에서 기록하고, 가격표 기반 평가금액과 포트폴리오 구성을 확인하며, 은퇴 시점의 필요자산을 추정하는 정적 웹앱이다.

라이브 앱: https://yjmoonn.github.io/assettrail/

## 최종 방향

- Firebase Auth와 Firestore로 자산, 히스토리, 은퇴 설정을 사용자별로 동기화한다.
- GitHub Pages로 정적 배포한다.
- GitHub Actions가 하루 1회 `prices.json`을 생성한다.
- 국내 주식, ETF, ETN은 KRX 가격표를 자동 생성해 일별 종가를 사용한다.
- 미국 주식, ETF는 yfinance 일별 종가를 사용한다.
- 앱은 `prices.json`을 읽어 평가금액과 손익을 계산한다.
- `CASH`와 `MANUAL`만 사용자가 입력한 수동 평가금액을 사용한다.
- 대량등록, 환율/통화 직접 입력, Alpha Vantage 의존성은 제거한다.

## 핵심 사용자 흐름

1. 사용자는 GitHub Pages 앱을 연다.
2. Google 로그인 전에는 로컬 저장소로 사용할 수 있고, 로그인 후에는 Firestore에 동기화된다.
3. 자산은 `KRX`, `US`, `CASH`, `MANUAL` 중 하나로 등록한다.
4. 같은 종목이라도 계좌가 다르면 별도 자산으로 등록한다.
5. 가격이 있는 `KRX`와 `US` 자산은 `prices.json` 기준으로 평가금액과 손익을 자동 계산한다.
6. `CASH`와 `MANUAL`은 사용자가 입력한 평가금액을 그대로 사용한다.
7. 사용자는 조회 기록을 저장해 자산 히스토리를 남긴다.
8. 매수, 매도, 리밸런싱, 관찰 판단은 매매일지에 남겨 나중에 복기한다.
9. 은퇴 시뮬레이터에서 현재 자산, 월 투자금, 은퇴 후 월지출, 수익률/물가 가정을 바꿔 필요자산을 확인한다.

## 자산 유형

| 유형 | 사용 대상 | 평가 방식 |
|---|---|---|
| `KRX` | 국내 주식, ETF, ETN | 수량 x 원화 종가 |
| `US` | 미국 주식, ETF | 수량 x 달러 종가 x USD/KRW |
| `CASH` | 현금성 자산 | 수동 평가금액 |
| `MANUAL` | 적금, 예금, 펀드, 기타 자동시세가 없는 금융상품 | 수동 평가금액 |

## 계좌/상품 분석 기준

포트분석은 자산별 평가금액을 계산한 뒤 아래 기준으로 요약한다.

| 분석 | 기준 |
|---|---|
| 계좌 분석 | 계좌명, 자산명, 메모의 키워드 |
| 계좌별 | 사용자가 입력한 계좌명 |
| 상품 유형 분석 | `prices.json`의 `kind` 또는 자산 유형 |
| 국내/해외 비중 | `KRX`, `US`, 기타 유형 |

현재 계좌 분석 규칙은 다음과 같다.

| 키워드 | 분류 |
|---|---|
| `적금`, `청약` | 적금 |
| `연금`, `IRP`, `퇴직`, `개인형퇴직연금`, `확정기여형`, `DC`, `DC형` | 연금계좌 |
| 그 외 계좌명이 있는 자산 | 일반계좌 |
| 계좌명이 없는 자산 | 계좌 미지정 |

주택청약저축은 `청약` 키워드 때문에 적금으로 분류한다. IRP와 DC는 연금계좌로 분류한다.

## 은퇴 시뮬레이터 가정

은퇴 시뮬레이터는 다음 가정으로 계산한다.

- `은퇴 후 월지출`은 현재 물가 기준 입력값이다.
- 은퇴 첫해 연 지출은 `은퇴 후 월지출 x 12`를 은퇴 시점 물가로 환산한 값이다.
- `은퇴 시점 필요 자산`은 은퇴 이후 더 이상 매월 추가투자금이 들어오지 않는다는 가정으로 계산한다.
- `매월 추가투자`는 현재부터 은퇴 시점까지 필요한 수익률 계산에만 반영된다.
- 은퇴 이후에는 남은 자산이 `은퇴 후 연수익률`로 운용되면서 생활비가 인출되는 구조다.

## 보안 원칙

- Firebase 웹 설정값은 브라우저 앱에서 공개될 수 있는 값이다.
- 실제 데이터 보호 경계는 Firestore Rules다.
- 사용자 자산 데이터는 `users/{uid}/financeData/primary`에 저장하며, 본인만 읽고 쓸 수 있다.
- 공유 가격 요청 문서 `priceRequests/us`는 GitHub Actions가 미국 티커를 수집하기 위해 사용한다.
- `.env`, 서비스 계정 JSON, private key, Firebase Admin 인증 정보는 커밋하지 않는다.
