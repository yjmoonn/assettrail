# AssetTrail 과거 할 일 목록

이 문서는 현재 AssetTrail 기능을 기준으로, 앱의 목표에 맞게 강화하거나 추가하면 좋은 작업을 우선순위로 정리한다.

## 우선순위 기준

| 우선순위 | 의미 |
|---|---|
| P0 | 데이터 손실, 계산 신뢰도, 로그인/동기화처럼 먼저 안정화해야 하는 항목 |
| P1 | 사용성이 크게 좋아지거나 포트폴리오 분석 품질을 올리는 항목 |
| P2 | 있으면 좋은 확장 기능 또는 운영 편의 기능 |

## 구현 상태

| 순서 | 항목 | 상태 |
|---:|---|---|
| 1 | 클라우드 동기화 충돌 방지 | 완료 |
| 2 | 삭제/비우기 작업 안전장치 강화 | 완료 |
| 3 | 가격표 신뢰도 표시 | 완료 |
| 4 | 계좌/상품 분류를 사용자가 직접 지정 | 완료 |
| 5 | MANUAL 자산 세부 유형 추가 | 완료 |
| 6 | 자산 원장 정렬/필터 강화 | 완료 |
| 7 | 히스토리 분석 강화 | 완료 |
| 8 | 포트폴리오 목표 비중과 리밸런싱 | 완료 |
| 9 | 은퇴 시뮬레이터 시나리오 저장 | 완료 |
| 10 | 내보내기/가져오기 검증 강화 | 완료 |

구현 메모:

- 계좌/상품 분류, MANUAL 세부 유형, 목표 비중, 은퇴 시나리오는 기존 데이터와 호환되도록 기본값을 둔다.
- 삭제/비우기 작업은 확인창과 즉시 되돌리기 알림을 함께 사용한다.
- 가격표 상태는 생성 시각, 오류 수, 환율 상태를 화면에 노출한다.
- 동기화 충돌은 클라우드 데이터와 현재 화면 데이터가 다를 때 사용자가 가져오기/올리기를 선택하게 한다.

## P0

### 1. 클라우드 동기화 충돌 방지

현재는 Firestore와 로컬 데이터가 동기화되지만, 여러 기기에서 동시에 수정했을 때 어떤 쪽이 최신인지 사용자가 알기 어렵다.

작업:

- 마지막 저장 시각을 화면에 표시한다.
- 클라우드 데이터와 로컬 데이터가 모두 변경된 경우 덮어쓰기 전에 안내한다.
- `Sync` 버튼을 눌렀을 때 가져오기/올리기 중 무엇이 일어났는지 더 명확히 보여준다.

완료 기준:

- 사용자가 “내 데이터가 언제 저장됐는지”와 “지금 화면이 클라우드와 맞는지”를 알 수 있다.
- 충돌 가능성이 있을 때 무심코 데이터를 덮어쓰지 않는다.

### 2. 삭제/비우기 작업 안전장치 강화

자산 삭제와 히스토리 비우기는 되돌리기 어렵다. 현재 확인창은 있지만, 장기적으로 더 안전한 삭제 UX가 필요하다.

작업:

- 히스토리 비우기 확인 문구를 더 명확히 한다.
- 자산 삭제 시 삭제 대상 이름, 계좌, 평가금액을 확인창에 함께 보여준다.
- 가능하면 삭제 직후 “되돌리기” 기능을 제공한다.

완료 기준:

- 사용자가 삭제되는 데이터의 범위를 명확히 알 수 있다.
- 실수로 누른 삭제를 즉시 되돌릴 수 있다.

### 3. 가격표 신뢰도 표시

가격표는 앱 계산의 핵심이다. 현재 가격 날짜와 가격 대기 상태를 보여주지만, 오래된 가격표나 일부 실패를 더 잘 드러낼 필요가 있다.

작업:

- `prices.json` 생성일이 오래됐을 때 경고를 표시한다.
- 가격 실패 종목을 자산 원장 상단에 더 구체적으로 보여준다.
- 미국 티커가 가격 요청 목록에 올라간 경우 “다음 가격표 생성 후 반영” 상태를 표시한다.

완료 기준:

- 사용자가 평가금액이 최신 가격인지 바로 판단할 수 있다.
- 가격 대기 상태가 “왜 대기인지”와 “언제 해결될지”를 알려준다.

## P1

### 4. 계좌/상품 분류를 사용자가 직접 지정

현재 계좌 분석은 `적금`, `청약`, `IRP`, `DC` 같은 키워드 규칙으로 분류한다. 편리하지만 예외가 생길 수 있다.

작업:

- 자산 등록/수정 화면에 `계좌 분류` 선택값을 추가한다.
- 기본값은 자동 추정으로 두되, 사용자가 직접 `일반계좌`, `연금계좌`, `적금`, `계좌 미지정` 등을 지정할 수 있게 한다.
- 기존 자산은 현재 키워드 기반 자동 분류를 유지한다.

완료 기준:

- 사용자가 원하는 기준으로 계좌 분석을 보정할 수 있다.
- 기존 저장 자산은 손상되지 않는다.

### 5. MANUAL 자산 세부 유형 추가

현재 적금, 예금, 펀드, 보험성 상품, 기타 수동 자산이 모두 `MANUAL`로 묶인다. 포트분석에서 더 잘 나누려면 세부 유형이 필요하다.

작업:

- `MANUAL` 자산에 세부 유형을 추가한다.
- 후보: `적금`, `예금`, `펀드`, `보험`, `기타`.
- 포트분석의 상품 유형 분석에서 세부 유형을 반영한다.
- 수동평가 자산에는 “평가금액 직접 갱신 필요” 안내를 표시한다.

완료 기준:

- 적금과 펀드가 같은 `수동평가`로만 보이지 않는다.
- 사용자가 수동평가 자산의 최신성 책임을 이해할 수 있다.

### 6. 자산 원장 정렬/필터 강화

현재 검색과 유형 필터는 있지만, 보유 자산이 많아지면 계좌별/평가금액별로 보기 어렵다.

작업:

- 열 기준 정렬을 추가한다.
- 계좌 필터를 추가한다.
- 가격 대기 자산만 보기 필터를 추가한다.
- 평가손익 양수/음수 필터를 추가한다.

완료 기준:

- 사용자가 문제 있는 자산이나 특정 계좌 자산을 빠르게 찾을 수 있다.

### 7. 히스토리 분석 강화

현재 히스토리는 조회 시점별 총자산 흐름을 보여준다. 다음 단계로 기간별 비교와 개별 기록 관리가 필요하다.

작업:

- 조회 기간 선택 필터를 추가한다.
- 히스토리 개별 삭제 기능을 추가한다.
- 특정 조회 기록에 메모를 남길 수 있게 한다.
- 직전 대비, 첫 조회 대비 외에 선택 기간 대비 변화를 보여준다.

완료 기준:

- 사용자가 “최근 1개월”, “올해”, “특정 기간”의 자산 변화를 확인할 수 있다.

### 8. 포트폴리오 목표 비중과 리밸런싱

현재 포트분석은 현재 비중을 보여준다. 사용자가 목표 비중을 정하면 다음 매수/조정 판단이 쉬워진다.

작업:

- 국내/해외, 주식/ETF/현금/수동평가 기준 목표 비중을 설정한다.
- 현재 비중과 목표 비중의 차이를 보여준다.
- 부족/초과 금액을 표시한다.

완료 기준:

- 사용자가 어느 자산군을 더 사거나 줄여야 하는지 판단할 수 있다.

## P2

### 9. 은퇴 시뮬레이터 시나리오 저장

현재 은퇴 시뮬레이터는 하나의 입력값만 저장한다. 여러 가정을 비교하려면 시나리오 저장이 필요하다.

작업:

- 보수적/기본/성장형 외에 사용자 시나리오 저장을 지원한다.
- 물가상승률, 은퇴 후 수익률, 월지출 변화에 따른 민감도 표를 제공한다.
- 현재 자산 중 연금계좌만 따로 반영하는 옵션을 검토한다.

완료 기준:

- 사용자가 은퇴 가정 여러 개를 저장하고 비교할 수 있다.

### 10. 내보내기/가져오기 검증 강화

JSON 내보내기/가져오기는 백업에 유용하지만, 잘못된 파일을 넣었을 때 안내가 더 필요하다.

작업:

- 가져오기 전에 요약 미리보기를 보여준다.
- 가져올 자산 수, 히스토리 수, 은퇴 설정 포함 여부를 표시한다.
- 가져오기 전 현재 데이터 백업을 권장한다.

완료 기준:

- 사용자가 어떤 데이터가 들어오는지 확인한 뒤 가져올 수 있다.

### 11. 운영 상태 대시보드

가격표와 배포는 GitHub Actions에 의존한다. 운영 상태를 앱 안에서 더 쉽게 확인하면 문제 파악이 빨라진다.

작업:

- 앱에 가격표 생성 시각, 가격표 오류 수, 환율 날짜를 요약 표시한다.
- 운영 문서에 수동 가격표 생성 절차를 더 자세히 적는다.
- GitHub Actions 실패 시 확인할 체크리스트를 정리한다.

완료 기준:

- 가격표가 이상할 때 사용자가 원인을 빠르게 좁힐 수 있다.

### 12. 접근성과 모바일 사용성 점검

현재 UI는 반응형 구조가 있지만, 장기 사용을 위해 입력 폼과 차트 접근성을 더 다듬을 수 있다.

작업:

- 모바일에서 자산 추가/수정 폼의 입력 순서를 점검한다.
- 차트에 텍스트 대체 설명을 보강한다.
- 버튼 아이콘에 더 명확한 `aria-label`을 제공한다.

완료 기준:

- 모바일에서도 자산 등록과 조회가 답답하지 않다.
- 차트 내용을 스크린리더 사용자도 이해할 수 있다.
