# AssetTrail 제품 경험 리디자인

## 요약

AssetTrail은 매일 수익률만 보는 앱도, 투자 추천 도구도 아니다. 현재 자산, 자산 변화, 투자 판단, 포트폴리오 균형, 히스토리, 은퇴 목표를 하나의 흐름으로 연결하는 개인 투자 원장이다.

제품 경험의 핵심 흐름은 아래 순서다.

```text
현재 상태 -> 변화 -> 판단 -> 포트폴리오 -> 장기 목표
```

현재 앱은 기능 깊이가 충분하지만, 사용자는 여러 도구가 길게 나열된 화면처럼 느낄 수 있다. 다음 제품 단계는 기능 묶음을 사용자의 투자 여정으로 재구성하는 것이다.

## 정보 구조

권장 화면 구조:

| 화면 | 역할 |
|---|---|
| 대시보드 | 오늘의 상태, 변화, 확인할 항목 |
| 자산 | 자산 원장, 매수, 매도, 자산 상세 |
| 일지 | 투자 판단, 실현손익, 사후 복기 |
| 포트폴리오 | 배분, 목표 비중 차이, 리밸런싱 힌트 |
| 목표 | 히스토리, 은퇴 계획 |
| 설정 | 클라우드 동기화, 가격표, 가져오기/내보내기, 위험 작업 |

구현 관점의 기능 나열보다 사용자가 다음 질문에 답하기 쉬운 구조를 우선한다.

- 지금 내 자산 상태는 어떤가?
- 최근 무엇이 바뀌었나?
- 확인하거나 기록해야 할 판단은 무엇인가?
- 목표 비중과 얼마나 차이가 나는가?
- 장기 목표에 비해 현재 속도는 충분한가?

## 우선순위

### P0

- 대시보드가 즉시 중요한 질문에 답하게 한다.
- 모바일 내비게이션을 명확하게 한다.
- 사용자 데이터와 Firestore 사용자별 분리 구조를 깨지 않는다.

### P1

- 동기화, 가져오기/내보내기, 가격 갱신 같은 운영 액션을 설정 또는 조용한 상태 영역으로 옮긴다.
- 대시보드에 “오늘 확인할 것” 흐름을 만든다.
- 자산 상세 흐름을 추가해 원장 행에 모든 정보를 밀어 넣지 않는다.
- 색상, 카드, 배지, 표 액션, 모달, 토스트를 더 일관된 시각 언어로 정리한다.

### P2

- 장식적 스타일을 줄이고 데이터 해석에 도움이 되는 표현을 우선한다.
- 접근성, 포커스 이동, `aria-live`, `aria-current` 같은 세부 품질을 보강한다.
- 차트와 데이터 시각화가 앱 토큰과 글꼴을 쓰도록 정리한다.

## 화면별 방향

### 대시보드

대시보드는 앱의 출발점이다. 모든 기능을 보여주는 곳이 아니라 사용자가 지금 봐야 할 상태를 요약하는 곳이다.

- 총자산, 변화, 가격표 상태, 최근 기록을 빠르게 보여준다.
- 국내/해외/현금/수동 비중은 목표 대비 차이를 읽기 쉽게 표현한다.
- 최근 투자 기록은 자산명과 행동이 정확하게 보여야 한다.
- 빈 상태에서도 다음 행동이 자연스럽게 보이게 한다.

### 자산

자산 화면은 원장 역할을 한다.

- 같은 티커라도 계좌가 다르면 별도 행으로 관리한다.
- `KRX`, `US`, `CASH`, `MANUAL`의 평가 기준이 명확해야 한다.
- 매수, 매도, 일지 작성 같은 행동은 찾기 쉬워야 한다.
- 모바일에서는 행이 너무 많은 정보를 한 줄에 밀어 넣지 않게 한다.

### 일지

일지는 투자 추천이 아니라 사용자 판단의 기록과 복기를 위한 공간이다.

- 매수, 매도, 리밸런싱, 관찰 판단을 구분한다.
- 투자 이유, 틀릴 수 있는 이유, 사후 복기 메모를 남길 수 있어야 한다.
- 자산 액션에서 일지 작성 화면으로 자연스럽게 이동해야 한다.

### 포트폴리오

포트폴리오 화면은 단순 비중 표시보다 목표와의 차이를 읽기 쉽게 만드는 것이 중요하다.

- 계좌별, 상품 유형별, 국내/해외 비중을 비교한다.
- 목표 비중 대비 초과/부족을 방향과 숫자로 표시한다.
- 차트는 장식보다 비교와 판단을 돕는 데 집중한다.

### 목표

목표 화면은 히스토리와 은퇴 시뮬레이터를 연결한다.

- 과거 스냅샷으로 자산 변화를 확인한다.
- 현재 자산, 월 투자금, 은퇴 후 월지출, 수익률, 물가 가정을 바꿔 필요자산을 계산한다.
- 계산 결과는 사용자가 다음에 조정할 변수를 알 수 있게 보여준다.

### 설정

설정 화면은 운영 액션을 모으는 곳이다.

- 로그인, 로그아웃, 동기화, 가격표 상태, 가져오기/내보내기를 정리한다.
- 위험 작업은 명확한 확인을 요구한다.
- 상단바와 설정에 같은 기능이 중복되어 사용자를 헷갈리게 하지 않는다.

## 시각 원칙

- 금융 데이터가 중심이므로 차분하고 일관된 화면 밀도를 유지한다.
- 카드 radius, 배지, 버튼, 표 액션의 스타일을 통일한다.
- 모바일에서는 하단 내비게이션과 주요 액션이 충돌하지 않게 한다.
- 텍스트가 버튼이나 카드 안에서 잘리지 않게 한다.
- 차트는 앱 색상 토큰과 글꼴을 사용한다.
- 장식적 효과보다 데이터의 비교 가능성과 신뢰감을 우선한다.

## 디자인 검토 방식

Figma 대신 실제 정적 앱을 기준으로 검토한다.

- 로컬 정적 서버로 앱을 연다.
- 데스크톱 1440px, 1280px을 확인한다.
- 모바일 390px, 430px을 확인한다.
- 주요 화면의 로그인 전/후, 빈 상태, 가격 대기 상태를 확인한다.
- 자세한 기준은 `docs/DESIGN_REVIEW_GUIDE.md`를 따른다.
