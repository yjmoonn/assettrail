# 세션 기록 작성 규칙

이 폴더는 AI 작업 세션의 핵심 요약만 보관한다. 전체 대화 로그를 저장하지 않는다.

## 언제 작성하나

- 문서 구조, 데이터 구조, 배포 방식, 테스트 기준처럼 프로젝트 작업 방식이 바뀐 경우
- 여러 파일을 바꾸는 기능 작업을 완료한 경우
- 다음 세션에서 “왜 이렇게 했는지” 알아야 할 결정이 생긴 경우

오타 수정, 단순 문구 변경, 작은 스타일 조정처럼 맥락 보존 가치가 낮은 작업은 기록하지 않아도 된다.

## 파일 이름

```text
YYYY-MM-DD-짧은-작업명.md
```

예:

```text
2026-06-27-ai-folder-reorg.md
```

## 작성 형식

```md
# 작업 제목

## 목적
한두 문장으로 작업 의도를 적는다.

## 핵심 결정
- 결정만 짧게 적는다.

## 변경
- 실제로 바뀐 구조나 파일을 적는다.

## 검증
- 실행한 검증과 결과를 적는다.

## 다음 작업
- 다음 세션에서 이어서 보면 좋은 항목을 적는다.
```

## 작성 원칙

- 대화 전문을 복사하지 않는다.
- 코드 diff를 길게 붙이지 않는다.
- 결정, 변경, 검증, 남은 작업만 남긴다.
- 최신 진행 상태는 `docs/HANDOFF_NOTES.md`에 짧게 반영한다.
