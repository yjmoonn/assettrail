# 포트폴리오 분석 파일럿

## 목적

AssetTrail 원장을 단일 기준으로 사용해 공통 포트폴리오 분석을 생성하고, 영재 계정 검증 후 다른 사용자에게 확대할 수 있는 첫 기반을 만든다.

## 핵심 결정

- 별도 `분석` 탭에서 사용자가 직접 실행한다.
- 현재 원장과 내보낸 JSON을 모두 지원하되, JSON 분석은 현재 원장을 교체하지 않는다.
- 계좌가 달라도 같은 티커는 하나의 경제적 포지션으로 합산한다.
- 현금흐름·ETF 구성·벤치마크가 없으면 TWR·XIRR·ETF 투시·위험조정 성과를 추정하지 않는다.
- 분석 이력은 최근 12회만 유지한다.
- PDF는 브라우저가 아니라 Firebase 인증을 검증하는 Cloud Run 서비스에서 8페이지로 생성한다.

## 변경

- `analysis-engine.js`: JSON 분석 스키마 v1과 결정론적 계산 엔진 추가.
- `index.html`, `app.js`, `styles.css`: 분석 탭, 실행, JSON 파일럿, 신뢰도·노출·집중도·성과·이력 화면 추가.
- `services/analysis-api/`: 사용자별 이력 저장과 PDF 생성 API 추가.
- `firestore.rules`: 원장·분석설정·분석결과 권한을 분리하고 분석결과의 클라이언트 쓰기 차단.
- 분석 엔진·화면·보고서·Rules 테스트 추가.
