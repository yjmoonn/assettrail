# 2026-07-09 성능·Firestore 비용 최적화

## 목적
체감 속도 + 코드 정리 + Firebase 비용 절감 종합 최적화. 8단계(Phase 1~8), 커밋 e94de0b~2bb0782.

## 핵심 결정
- **최우선 수정은 버그였다**: `addEventListener("input", render)`가 InputEvent를 `syncCloud=true`로 해석 → 은퇴·목표 비중 입력의 키 입력 1회마다 Firestore setDoc 1회. 이 한 줄이 비용 최대 항목.
- **쓰기 절감 3종 세트**: ① 입력 확정(change) 시에만 저장 ② fingerprint 비교로 무변경 스킵 ③ 2초 debounce 배칭 + 이탈 시 flush. localStorage는 즉시 저장 유지라 유실 시 기존 충돌 감지로 복구.
- **부분 렌더링 전제 조건**: 렌더 함수가 입력값을 state에 쓰는 결합(`renderRetirement`→`saveRetirementInputs` 등)을 먼저 끊어야 함(Phase 4). 안 끊으면 test:cloud 깨짐.
- **지연 렌더링 방식**: 뷰→렌더 함수 매핑 + dirty Set. render는 활성 뷰만, 나머지는 화면 전환 시. 부팅은 전체 1회.
- **테스트 계약 변경**: jsdom 테스트가 숨겨진 뷰의 DOM을 검증하려면 해당 뷰 전환 클릭이 선행돼야 함. jsdom에 `scrollIntoView`가 없어 목 추가. debounce는 `window.assetTrailCloudPushDelayMs=0` 오버라이드.
- **스코프 아웃**: app.js 분할(테스트가 단일 파일 전제), innerHTML 전면 전환, 다크모드 토큰 정리, Phase 9(팔레트 캐시 — 효과 미미로 스킵).
